Capstone - Kellanova Demand Forecasting Project

This capstone project focused on building and evaluating demand forecasting models for weekly point-of-sale sales data in collaboration with Kellanova. Because the original data is protected under an NDA, only a generic copy of the project using dummy data is shared. The project compares several forecasting approaches, including SARIMAX, Prophet, NBEATS, NHits, LSTM, XGBoost, Chronos2, TSMixer, LightGBM, and DeepAR. The analysis focuses on forecast accuracy, provider-level performance, bias, volatility, and model comparison across a 26-week hold-out period.

Python numpy pandas matplotlib scikit-learn statsmodels Prophet XGBoost LightGBM PyTorch NeuralForecast Darts Chronos2 Jupyter Notebooks Conda
More details

Problem

This project addresses the challenge of forecasting weekly consumer demand using historical point-of-sale data and trade-related features. The goal was to identify models that could accurately predict future sales patterns across multiple providers while accounting for trends, seasonality, promotional activity, and trade distribution effects. Since the original data is protected under an NDA, the shared version of the project uses dummy data to demonstrate the structure, workflow, and modeling process without exposing protected information.

Approach

The project cleaned and aggregated weekly sales data by provider, engineered time-based and trade activity features, and tested multiple forecasting models over a consistent 26-week hold-out period. Models were compared using standardized metrics such as RMSE, MAPE, WMAPE, accuracy, bias, volatility ratio, and standard deviation of absolute percentage error.

Results & Impact

The results provided a structured comparison of several forecasting methods and highlighted differences in model performance across providers. The project showed that demand forecasting accuracy can vary depending on provider behavior, sales volatility, and model type. By incorporating metrics beyond basic error measurements, such as bias and volatility ratio, the analysis helped identify models that were not only accurate on average but also more stable and reliable. These findings can support future forecasting workflows, model selection, and business planning for demand-driven decision making.

Data Note

The original project data is protected under a non-disclosure agreement, so the shared version of this project only includes dummy data. This allows the general project structure, notebooks, modeling workflow, environment setup, and visualization outputs to be reviewed without exposing confidential data.

AWS Cloud Computing - NYC 311 Service Request Analysis and Modeling Project

This project analyzes NYC 311 service requests to understand complaint patterns and predict the expected time to resolution. It uses a modeling dataset derived from NYC Open Data and explores how agency, borough, complaint type, ZIP code, and time based features relate to `days_to_close`. The main outputs are exploratory analysis, a scikit-learn regression model, and a SageMaker Linear Learner workflow.

Python AWS pandas numpy scikit-learn boto3 SageMaker Jupyter Notebooks
More details

Problem

This project investigates which complaint and context features help explain how long NYC 311 complaints take to close. The goal is to understand the distribution of resolution times and build models that estimate `days_to_close` for stakeholder analysis.

Approach

The project begins by validating the raw complaint and agency data, then builds a modeling table with complaint, geography, and time based features. The notebooks explore the target distribution, split the data into train and test sets, and compare two modeling workflows: a scikit-learn random forest regressor and a SageMaker Linear Learner regression pipeline. Categorical variables are encoded, missing values are handled, and model performance is evaluated with MAE, RMSE, and R^2.

Results & Impact

The analysis shows that `days_to_close` is highly skewed toward immediate closures, with most complaints resolved on day 0 and a long tail of slower cases. The random forest model and the SageMaker Linear Learner both produce useful predictions, with the random forest performing best overall. These results provide a practical summary of complaint resolution patterns and a foundation for future modeling improvements.

Sakila SQL Revenue Analysis

This project analyzes rental revenue patterns using the Sakila database. It was adapted from a course project and revised into a standalone portfolio analysis. The analysis connects payment, rental, inventory, film, and category tables to evaluate which film genres generate the most revenue and how genre performance changes over time. The main outputs include SQL-based revenue summaries, monthly genre rankings, validation checks, an EXPLAIN query performance review, and a stakeholder-ready report table with a supporting visualization.

Python pandas SQL MySQL SQLAlchemy matplotlib Jupyter Notebooks Docker
More details

Problem

This project investigates how different film genres contribute to rental revenue in the Sakila database. The goal is to identify top performing genres, compare revenue trends across monthly periods, and create a clear report that could support business decisions for a store manager or marketing analyst.

Approach

The project begins by connecting Python to a Dockerized MySQL Sakila database using SQLAlchemy. Multi table SQL joins are used to connect payment records with rentals, inventory, films, and categories. The analysis then applies grouped aggregations, HAVING filters, common table expressions, and window functions to summarize genre revenue, calculate monthly revenue share, and rank genres within each month. Validation queries are included throughout the notebook to check row counts, aggregation results, and ranking logic. An EXPLAIN query is also used to review query performance and understand join behavior.

Results & Impact

The analysis identifies which film genres contribute the most revenue and shows how genre performance changes by month. The final report table provides monthly revenue, percent of monthly revenue, and revenue rank by genre, making the results easier to interpret for business decision-making. The project demonstrates practical SQL skills including joins, aggregation, CTEs, window functions, validation, and basic performance review in a business analytics context.
